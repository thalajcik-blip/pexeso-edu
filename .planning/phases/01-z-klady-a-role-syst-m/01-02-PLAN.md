### Task 1: DB schema — `child_consents` table and `is_minor` column on `profiles`

**File:** `supabase/migrations/$(date +%Y%m%d%H%M%S)_gdpr_consent_schema.sql`

**What:** Create the `child_consents` table with RLS, add `is_minor` boolean to `profiles`, and add the privacy-by-default RLS policy on `profiles`.

**How:**

Create the migration file with this exact SQL:

```sql
-- GDPR-03: consent record table
CREATE TABLE IF NOT EXISTS child_consents (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  child_user_id   uuid NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  consent_version text NOT NULL DEFAULT 'v1',
  consented_at    timestamptz NOT NULL DEFAULT now(),
  ip_hash         text,
  UNIQUE(child_user_id, consent_version)
);

ALTER TABLE child_consents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "user_sees_own_consent" ON child_consents
  FOR SELECT USING (auth.uid() = child_user_id);

CREATE POLICY "user_inserts_own_consent" ON child_consents
  FOR INSERT WITH CHECK (auth.uid() = child_user_id);

-- GDPR-05: privacy-by-default flag
ALTER TABLE profiles ADD COLUMN IF NOT EXISTS is_minor boolean NOT NULL DEFAULT false;

-- GDPR-05: block public reads for minor profiles
-- Drop any conflicting all-public SELECT policy first if it exists
-- Then add the restricted policy:
CREATE POLICY "minor_profile_not_public"
  ON profiles FOR SELECT
  USING (
    id = auth.uid()
    OR is_minor = false
    OR EXISTS (
      SELECT 1 FROM profiles WHERE id = auth.uid() AND 'superadmin' = ANY(roles)
    )
  );
```

Apply via `supabase db push`.

Note: If an existing profiles SELECT policy conflicts (e.g., `USING (true)`), drop it first. Check existing policies in Supabase Dashboard under Authentication → Policies → profiles before running.

**How:**

Then, in `delete-account/index.ts`, find the deletion sequence (likely lines 30–40) and add these deletions BEFORE the `profiles` row deletion (because `child_consents` has a FK cascade, but `teacher_requests` and Storage do not):

```typescript
// Delete teacher_requests
await adminClient
  .from('teacher_requests')
  .delete()
  .eq('user_id', userId)

// Delete child_consents (also cascades from profiles FK, but explicit is safer)
await adminClient
  .from('child_consents')
  .delete()
  .eq('child_user_id', userId)

// Delete Storage objects in card-images bucket
// Path structure: {userId}/{filename} — confirm from BulkUploadModal.tsx
const { data: storageObjects } = await adminClient.storage
  .from('card-images')
  .list(userId)

if (storageObjects && storageObjects.length > 0) {
  const paths = storageObjects.map(obj => `${userId}/${obj.name}`)
  await adminClient.storage.from('card-images').remove(paths)
}
```
