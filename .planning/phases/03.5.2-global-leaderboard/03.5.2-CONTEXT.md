# Phase 03.5.2: Global Leaderboard - Context

**Gathered:** 2026-03-29
**Status:** Ready for planning

<domain>
## Phase Boundary

Táto fáza nahradí placeholder "Rebríček bude dostupný v ďalšej verzii." v existujúcom Rebríček tabe na `/leaderboard` funkčným globálnym leaderboardom — top 50 hráčov podľa celkového súčtu skóre, s time filtrom (týždeň/mesiac/celkovo), per-sada filtrom a zobrazením pozície prihláseného hráča.

**Čo táto fáza dodáva:**
1. Supabase query agregujúca celkové skóre hráčov zo `game_history` (GROUP BY user_id, SUM score)
2. Time filter: Celkovo (default) / Tento týždeň / Tento mesiac
3. Per-sada filter: dropdown so sadami — rank sa prepočíta pre vybranú sadu
4. Top 50 tabuľka: Rank, Avatar + username, Celkové skóre, Počet hier
5. Fixný riadok na konci pre prihláseného hráča ak nie je v top 50
6. Hodinový polling refresh (nie Realtime)
7. GDPR: hráči `age_group = 'under_16'` vylúčení z query

**Čo táto fáza NErobí:**
- Nerieši i18n routing ani lokalizované URL (to je Phase 3.5)
- Nerieši Daily Challenge leaderboard (to je Phase 4)
- Neimplementuje notifikácie pri novom osobnom rekorde

</domain>

<decisions>
## Implementation Decisions

### Ranking metrika
- **D-01:** Rank je určený celkovým súčtom skóre hráča (SUM score zo `game_history`)
- **D-02:** Rank je vždy vypočítaný zo všetkých hier (nie filtrovaný time filtrom) — time filter len zobrazuje zúžený pohľad ale poradie je konštantné
- **D-03:** Ak je aktívny per-sada filter, rank = SUM score len pre hry v danej sade — konzistentná metrika

### Time filter
- **D-04:** Tri možnosti: Celkovo (default) / Tento týždeň / Tento mesiac
- **D-05:** Default pri načítaní: "Celkovo"
- **D-06:** Time filter sa aplikuje na `played_at` v `game_history` — ovplyvňuje len aké záznamy vstupujú do agregácie pre rank pri per-sada pohľade. Pre globálny rank (D-02) time filter rank neovplyvňuje — iba filtruje zobrazené dáta.

  > Poznámka pre plannera: Zvážiť či time filter pri "Celkovo" rank len zobrazuje top hráčov podľa celoživotného súčtu alebo filtruje aj záznamy. D-02 hovorí rank = vždy všetky hry — time filter je teda len pre per-sada pohľad alebo je to UI element bez vplyvu na globálny rank. Implementuj ako: globálny rank = vždy všetky hry; time filter je reservovaný pre per-sada mód alebo budúce rozšírenie.

### Per-sada filter
- **D-07:** Dropdown zobrazuje všetky dostupné sady — hráč môže filtrovať rebríček na konkrétnu sadu
- **D-08:** Pri výbere sady sa rank prepočíta: SUM score pre hry v danej sade (WHERE set_slug = X alebo set_title = X)
- **D-09:** Default: žiadna sada (globálny rebríček)

### Pozícia hráča
- **D-10:** Top 50 hráčov zobrazených v tabuľke
- **D-11:** Ak je prihlásený hráč mimo top 50 — zobraziť fixný riadok na konci tabuľky (oddelený oddelovačom) s jeho aktuálnym rankom, avatrom a skóre
- **D-12:** Ak prihlásený hráč je v top 50 — jeho riadok sa zvýrazní (highlight)

### Stĺpce tabuľky
- **D-13:** Rank (#1, #2, ...) | Avatar + username | Celkové skóre | Počet hier

### Refresh
- **D-14:** Hodinový polling (`setInterval` 3 600 000ms) — nie Supabase Realtime

### GDPR
- **D-15:** Hráči s `profiles.age_group = 'under_16'` sú vylúčení z query (JOIN profiles + WHERE age_group != 'under_16')

</decisions>
