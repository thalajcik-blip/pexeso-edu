## Summary

Táto fáza stojí na solídnom základe: `game_history` tabuľka a `saveGameResult()` funkcia sú už implementované z predchádzajúcich fáz. CONTEXT.md predpokladá, že tieto neexistujú — to je nepresná premisa, ktorú musí planner zohľadniť. Primárna práca je: (1) vybudovať `/leaderboard` route s Tabs, (2) nainštalovať TanStack Table + shadcn table komponent, (3) vytvoriť GameResultsTable s filtrovacím toolbarom, (4) vytvoriť nový `GameResult` display typ pre leaderboard (odlíšiť od existujúceho `GameResult` v `gameService.ts`).

Kľúčový problém: existujúci `gameService.ts` ukladá `game_mode` ako `'pexequiz' | 'lightning'`, ale CONTEXT.md a brief predpokladajú hodnotu `'bleskovy_kviz'`. Toto je `game_mode` string mismatch — musí byť vyriešený pred alebo v rámci tejto fázy. DB constraint a frontend query musia používať rovnaké hodnoty.

`/leaderboard` route neexistuje v žiadnom súbore projektu — musí byť vytvorená. Routing v `main.tsx` je path-based if/else, nie React Router `<Route>` — leaderboard route vyžaduje pridanie vetvy do `main.tsx` a nový `LeaderboardPage` komponent. Tabs shadcn komponent nie je nainštalovaný.

**Primary recommendation:** Začať inštaláciou závislostí → `game_mode` migration fix → `LeaderboardPage` s Tabs → `GameResultsTable` s TanStack Table.

```typescript
const table = useReactTable({
  data,
  columns,
  getCoreRowModel: getCoreRowModel(),
  getSortedRowModel: getSortedRowModel(),
  getFilteredRowModel: getFilteredRowModel(),
  getPaginationRowModel: getPaginationRowModel(),
  onSortingChange: setSorting,
  onColumnFiltersChange: setColumnFilters,
  state: { sorting, columnFilters },
  initialState: { pagination: { pageSize: 20 } },
})
```

## Common Pitfalls

### Pitfall 1: GameResult type name collision
**What goes wrong:** `src/services/gameService.ts` exportuje `GameResult` interface (pre INSERT). Ak `src/types/gameResult.ts` exportuje iný `GameResult` (pre display), importy sa môžu pomýliť.
**Why it happens:** Obe veci sa volajú "GameResult" logicky ale majú inú shape.
**How to avoid:** Premenovať display typ na `GameResultRow` alebo použiť explicit alias pri importe: `import type { GameResult as GameResultRow } from '../types/gameResult'`.
**Warning signs:** TypeScript error "Property X does not exist" keď sa mieša display type so service type.

### Pitfall 2: TanStack filter reset pri prázdnom poli vs. undefined
**What goes wrong:** `table.getColumn('game_mode')?.setFilterValue([])` a `setFilterValue(undefined)` správajú sa rozdielne — prázdne pole môže byť interpretované ako "filter na prázdny zoznam" namiesto "žiadny filter".
**Why it happens:** Custom `filterFn: (row, id, value) => value.includes(...)` — prázdne pole `[].includes(x)` = vždy false = zobrazia sa 0 riadkov.
**How to avoid:** Pre "reset" filter nastaviť `undefined`, nie `[]`. Alebo explicitný check: `if (value.length === 0) return true` v custom filterFn.

### Pitfall 3: is_multiplayer boolean vs. string v Select filter
**What goes wrong:** Supabase vracia `is_multiplayer: boolean`. Select component vracia string `'true'`/`'false'`. Custom filterFn robí `value.includes(String(row.getValue(id)))` — ok, ale clearovanie cez `setFilterValue([])` musí tiež pristupovať cez string konverziu.
**How to avoid:** Konzistentne používať `String(boolean)` konverziu v oboch smeroch v `filterFn` a v Select `onValueChange`.

### Pitfall 4: Predpokladať `table.tsx` shadcn je nainštalovaný
**What goes wrong:** `import { Table } from '@/components/ui/table'` hodí module not found.
**How to avoid:** `npx shadcn@latest add table tabs` musí byť prvý krok Wave 0.

### Pitfall 5: profiles join — avatar_id môže byť null
**What goes wrong:** `profiles.avatar_id` môže byť null pre starých userov (onboarding ich nemusel nastaviť). `Avatar` komponent dostane `null` namiesto `number`.
**Why it happens:** `profiles(username, avatar_id)` join — avatar_id je nullable stĺpec.
**How to avoid:** `row.original.avatar_id ?? 0` alebo `?? 1` ako fallback. `Avatar` komponent zvláda `undefined/0` — volá `getAvatarDataUri(0)`.

   - Recommendation: Overiť v `LightningGame.tsx` a `WinModal.tsx` — oba komponenty sú zdieľané (nie len hostové). `saveGameResult` skontroluje `user.id` — non-host guests bez konta nebudú ukladať.

   - What's unclear: Má `profiles` stĺpec `age_group` alebo iný mechanizmus?
   - Recommendation: Pre MVP — ak RLS `teacher_reads_student_history` policy už existuje a je príliš permissive, nový `resultsService.ts` môže filtrovať podľa dostupného GDPR stĺpca.
