3. Export `getGameResults()` function:
```typescript
export async function getGameResults(opts: {
  mode?: 'pexequiz' | 'lightning'
  limit?: number
  userId?: string
}): Promise<GameResultRow[]>
```

Implementation:
- Import `supabase` from `./supabase`
- Import `GameResultRow` from `../types/gameResult`
- Build query: `supabase.from('game_history').select('id, played_at, user_id, set_slug, set_title, custom_deck_id, game_mode, score, quiz_correct, quiz_total, total_pairs, duration_sec, is_multiplayer, profiles!inner(username, avatar_id)')`
- If `opts.userId` is provided: add `.eq('user_id', opts.userId)` — personal history
- If `opts.userId` is NOT provided: this is global view (no user filter)
- If `opts.mode` is provided: add `.eq('game_mode', opts.mode)`
- Add `.order('played_at', { ascending: false })`
- Add `.limit(opts.limit ?? 100)`
- Map response rows to `GameResultRow[]`:
  - `username` = `row.profiles.username ?? 'Anonym'`
  - `avatar_id` = `row.profiles.avatar_id ?? 0`
  - `accuracy` = `row.quiz_total > 0 ? Math.round(row.quiz_correct / row.quiz_total * 100) : null`
  - `set_name` = `row.set_title ?? (row.set_slug ? (SET_NAMES[row.set_slug] ?? row.set_slug) : '—')`
- Return mapped array

**Per LB-01 GDPR note:** For MVP, the existing RLS policy is sufficient. The `profiles!inner` join naturally excludes rows where profile was deleted. Full GDPR under-16 filtering (via `age_group`) can be added as a follow-up if `profiles.age_group` column exists.
