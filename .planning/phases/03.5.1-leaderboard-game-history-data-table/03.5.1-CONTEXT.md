<domain>
## Phase Boundary

Táto fáza pridáva kompletnú infraštruktúru pre záznamy výsledkov hier a ich zobrazenie vo filtrovateľnej, zoraditeľnej Data Table na `/leaderboard` stránke.

**Čo táto fáza dodáva:**
1. Supabase `game_history` tabuľka — každá hra (solo + multiplayer, PexeQuiz + Bleskový kvíz) ukladá výsledok
2. `GameResult` TypeScript typ a `resultsService.ts` Supabase query
3. shadcn Data Table komponent (TanStack Table) s filtrovacím toolbarom
4. `/leaderboard` route rozšírená o Tabs: "Rebríček" (existujúci) | "Výsledky hier" (nový)
5. Integrácia do game engine — `saveGameResult()` volaná na konci každej hry

**Čo táto fáza NErobí:**
- Nebuiluje základnú `/leaderboard` route (tá je v Phase 3.5)
- Neimplementuje user profily (predpokladá existujúcu `profiles` tabuľku s `username` + `avatar_id`)
- Neimplementuje Daily Challenge leaderboard (to je Phase 4)

</domain>

<decisions>
## Implementation Decisions

### LB-01: Supabase schema — game_history tabuľka
- Tabuľka `game_history` s columns: `id uuid PK`, `user_id uuid FK → profiles.id`, `played_at timestamptz DEFAULT now()`, `game_mode text CHECK IN ('pexequiz','bleskovy_kviz')`, `score integer`, `quiz_correct integer`, `quiz_total integer`, `total_pairs integer`, `duration_sec integer`, `is_multiplayer boolean DEFAULT false`, `set_slug text`, `custom_deck_id uuid NULLABLE FK → custom_decks.id`
- RLS: `SELECT` povolený pre vlastníka (`auth.uid() = user_id`) + verejný globálny view (service role alebo verejný view bez PII); `INSERT` len pre autentifikovaného vlastníka
- GDPR: hráči pod 16 rokov (`profiles.age_group = 'under_16'`) sa nezobrazujú v globálnom leaderboarde — filtrovať cez view alebo query condition

### LB-02: GameResult TypeScript typ
- Súbor: `src/types/gameResult.ts`
- `accuracy` je computed na frontende: `quiz_total > 0 ? (quiz_correct / quiz_total * 100) : null`
- `set_name` je display-only string odvodený zo `set_slug` (lookup cez `SET_NAMES` konštantu)
- `username` a `avatar_id` pochádzajú z `profiles` join

### LB-03: resultsService.ts — Supabase query
- Súbor: `src/services/resultsService.ts`
- Funkcia `getGameResults({ mode?, limit = 100, userId? })` — ak `userId` undefined → globálne výsledky, inak vlastná história
- Join na `profiles(username, avatar_id)` vo Supabase query
- Order: `played_at descending` (default)
- Keď `userId` je undefined: filtrovať `under_16` profilů z výsledkov (GDPR)

### LB-04: Inštalácia závislostí
- `npm install @tanstack/react-table`
- `npx shadcn@latest add table`
- Existujúce shadcn komponenty (`Button`, `Input`, `Select`, `Badge`, `DropdownMenu`) sú už nainštalované — netreba reinštalovať

### LB-05: columns.tsx — definícia stĺpcov
- Súbor: `src/components/leaderboard/columns.tsx`
- Stĺpce: **Hráč** (Avatar 28px + username link `/profil/:username`), **Sada** (`SET_NAMES` lookup, filterable), **Mód** (Badge: `🃏 PexeQuiz` default | `⚡ Bleskový kvíz` secondary), **Skóre** (bold, `#F5C400`), **Presnosť** (mini progress bar + %, zelená ≥90%, žltá ≥60%, červená <60%), **Typ** (Badge outline: Solo | Online), **Dátum** (`cs-CZ` locale, muted)
- Sortable columns: Hráč, Skóre, Presnosť, Dátum (cez ArrowUpDown button ghost)
- `SET_NAMES` konštanta: `{ vlajky: 'Vlajky', zviratka: 'Zvířátka', 'ovoce-zelenina': 'Ovoce & zelenina', povolani: 'Povolání' }` + custom decky zobrazia `set_name` priamo

### LB-06: GameResultsTable.tsx — DataTable komponent
- Súbor: `src/components/leaderboard/GameResultsTable.tsx`
- Default sort: `played_at` descending
- Filters toolbar: Input "Hľadaj sadu..." (filterFn: includesString na `set_slug`), Select "Všetky módy / 🃏 PexeQuiz / ⚡ Bleskový kvíz" (filterFn: array includes), Select "Solo / Online" (filterFn: string boolean)
- Reset button: zobrazí sa len keď `columnFilters.length > 0`
- Pagination: 20 riadkov/stránku, `← Predchádzajúca` / `Ďalšia →` buttons, "X / Y" stránok
- Loading state: skeleton rows (shadcn Skeleton komponent)
- Empty state: "Žiadne výsledky" centered
- Styling: dark table s `border: 1px solid rgba(255,255,255,0.08)`, `borderRadius: 12`

### LB-07: /leaderboard route — Tabs integrácia
- Existujúci leaderboard content (top-50 rebríček) sa zabalí do `<TabsContent value="ranking">`
- Nový `<TabsContent value="results">` obsahuje `<GameResultsTable />`
- shadcn `Tabs` komponent (`npx shadcn@latest add tabs` ak chýba)
- Default tab: `"ranking"` (existujúci obsah je primárny, tabuľka je sekundárny pohľad)
- Tabs labels: "Rebríček" | "Výsledky hier"

### LB-08: Game engine integrácia — ukladanie výsledkov
- Nová funkcia `saveGameResult(result: Omit<GameResult, 'id' | 'played_at' | 'accuracy' | 'username' | 'avatar_id' | 'set_name'>)` v `src/services/resultsService.ts`
- Volá sa z `gameStore.ts` pri prechode do `game_over` / `lightning_results` fázy
- Len pre prihlásených hráčov (`supabase.auth.getUser()` check pred insertom)
- Solo hra: `user_id` = aktuálny hráč, `is_multiplayer = false`
- Online multiplayer: každý hráč ukladá vlastný výsledok lokálne (nie host za ostatných)
- Error handling: fire-and-forget (try/catch, neblokuje UI)
- PexeQuiz score = `pairs + quizCorrect`; Bleskový kvíz score = `lightningPlayerStats[myPlayerId].correct`

### LB-09: Avatar komponent
- Nový komponent `src/components/Avatar.tsx` — zobrazuje avatar podľa `avatarId` čísla
- Ak `profiles` tabuľka nemá `avatar_id` stĺpec: zobraziť initials fallback (prvé písmeno `username`)
- Veľkosť: `size` prop (default 28px), kruhový tvar
- Pre MVP: emoji avatary alebo farebné kruhy s iniciálou — nezávislý na externých obrázkoch

### Claude's Discretion
- Konkrétna implementácia Avatar komponentu (emoji set alebo farba+iniciála)
- Či je `profiles.avatar_id` už prítomný alebo treba pridať migráciu
- Poradie importov a presná štruktúra súborov
- Debounce na filter inpute (odporúčané 150ms pre UX)
- Či použiť `useCallback` na filter handlers (performance optimization)

</decisions>

```ts
export type GameResult = {
  id: string
  played_at: string
  username: string

  set_name: string
  set_slug: string

  quiz_correct: number

  accuracy: number | null
  total_pairs: number
  duration_sec: number
  is_multiplayer: boolean
}
```

## Deferred Ideas

- `/profil/:username` stránka (linky z Hráč stĺpca) — Phase 3.5 alebo neskôr

- Per-sada leaderboard filter (top hráči pre konkrétnu sadu) — Phase 3.5 má vlastný Plan 3 pre top-50 leaderboard
- Realtime subscription na game_history — polling postačuje (60s interval ako v Phase 4)
- Notifikácia pri novom osobnom rekorde
